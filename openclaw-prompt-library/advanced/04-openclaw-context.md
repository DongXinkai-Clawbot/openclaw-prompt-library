# 🧠 OpenClaw 上下文优化

> 针对 OpenClaw 记忆系统和长上下文的优化 Prompt

---

## OpenClaw 记忆架构

```
┌─────────────────────────────────────────┐
│           OpenClaw Memory               │
├─────────────────────────────────────────┤
│ 1. Workspace Files                      │
│    - AGENTS.md (角色定义)               │
│    - SOUL.md (人格)                     │
│    - USER.md (用户信息)                  │
│    - HEARTBEAT.md (任务)                │
│                                          │
│ 2. Session Memory                       │
│    - 当前会话历史                        │
│    - 最近 N 条消息                       │
│                                          │
│ 3. Long-term Memory (lancedb-pro)       │
│    - 向量存储                            │
│    - BM25 检索                          │
│    - 混合搜索                           │
│                                          │
│ 4. Daily Logs (memory/)                 │
│    - 压缩的历史日志                      │
└─────────────────────────────────────────┘
```

## 记忆锚点 Prompt

### 执行前检索

```
【任务】[任务描述]

【开始前 - 必做】
1. 检索相关历史：
   - 使用 memory_recall 搜索关键词：[任务相关关键词]
   - 检查 ~/clawd/memory/ 中的相关文件
   - 查看 workspace 中的上下文文件

2. 检查任务状态：
   - HEARTBEAT.md 中的待办
   - TASKS/ 目录中的任务文件

3. 准备上下文：
   - 只加载相关的 AGENTS.md 片段
   - 使用最小必要的历史消息
```

### 执行中记录

```
【记录规范】
每个关键步骤后：
1. 记录决策到 session memory
2. 更新相关文件（如需要）
3. 标记已完成的任务

【格式】
```
=== [时间戳] ===
[决策/操作描述]
原因：[为什么这样做]
结果：[产出]
下一步：[待办]
===
```
```

### 执行后保存

```
【任务完成 - 保存】

1. 摘要保存到 long-term memory：
   - 任务：[描述]
   - 结果：[成功/失败/部分成功]
   - 学到：[经验教训]
   - 关联：[相关任务/文件]

2. 清理：
   - 删除临时文件
   - 关闭无用连接
   - 释放资源

3. 报告：
   - 简洁汇报结果
   - 说明下一步建议
```

## 长上下文处理

### 分块处理策略

```
【背景】
当前上下文：[N] tokens
限制：[M] tokens

【策略】
1. 如果上下文 > 80% 限制：
   - 压缩历史消息（保留关键决策）
   - 卸载不相关文件
   - 使用 summary 代替详情

2. 分块执行：
   - 拆分为子任务
   - 每块独立执行
   - 结果合并

【压缩示例】
# 原始 (500 tokens)
[完整的对话历史]

# 压缩后 (50 tokens)
[2026-03-02 14:00-14:30]
- 任务：分析 Crypto 套利机会
- 决策：使用 Uniswap + Curve 三角套利
- 结果：发现机会，利润率 0.5%
- 下一步：需要更多资金
```

### 关键信息提取

```
【提取规则】
从长上下文中提取：
1. 决策点：做出的关键选择
2. 约定：与用户的约定
3. 偏好：用户的偏好/习惯
4. 错误：犯过的错误及修复
5. 上下文：当前任务背景

【格式】
```
=== KEY CONTEXT ===
decisions:
- [决策1]
- [决策2]

agreements:
- [约定1]

preferences:
- [偏好1]

errors_fixed:
- [错误及修复]

current_task:
- [当前任务背景]
===
```
```

## 动态 Prompt 优化

### 上下文感知模板

```
根据当前上下文长度，动态调整 Prompt：

【< 50% 上下文】
[标准模式]

【50-80% 上下文】
[精简模式]
- 减少示例
- 压缩背景
- 关键约束

【> 80% 上下文】
[最小模式]
- 只保留任务和核心约束
- 使用缩写
- 引用而非复制

【> 95% 上下文】
- 停止新任务
- 完成当前任务
- 触发压缩
```

### 自适应提示词

```
你是一个 AI 助手。

【上下文评估】
- 已用 tokens：[N]/[M] ([X]%)
- 当前任务复杂度：[低/中/高]

【自动调整】
根据评估结果：
1. 如果复杂度高 + 上下文足：
   - 使用完整模板
   
2. 如果复杂度低 + 上下文紧：
   - 使用最小模板
   
3. 如果上下文 > 90%：
   - 提示用户清理
   - 建议压缩
```

## OpenClaw 特定技能

### heartbeat 任务

```
【HEARTBEAT.md 使用规范】

读取规则：
1. 启动时读取 HEARTBEAT.md
2. 检查待办任务
3. 按优先级执行
4. 更新任务状态

更新规则：
1. 完成后标记 [完成]
2. 添加新任务格式：
   ```
   - [ ] [任务描述] #[优先级]
   ```
3. 删除已取消任务

【模板】
```
## 待办
- [ ] 任务1 #high
- [x] 任务2 #medium

## 进行中
- 任务3 [=======     ] 70%

## 完成
- 任务4 ✅
```
```

### 多 Agent 协作

```
【Agent 分工】

1. 规划 Agent：
   - 分析需求
   - 拆解任务
   - 分配给执行 Agent

2. 执行 Agent：
   - 执行具体任务
   - 报告进度
   - 处理错误

3. 验证 Agent：
   - 检查结果
   - 提出改进
   - 确认完成

【消息格式】
```
TO: [Agent名称]
FROM: [发送者]
TASK: [任务描述]
CONTEXT: [相关上下文]
DEADLINE: [截止时间]

[任务详情]
